ออกแบบให้โหลดไว

ใช้ SVG/ระบบฟอนต์ของเครื่อง, lazy-loading, และโค้ด JS น้อย เพื่อลด TTI และ CLS

ตารางสรุปผลการคัด, สถิติความมั่นใจ, อัตราคัดทิ้ง/ผ่าน สำหรับ QC

รองรับหน้างาน

ดีไซน์ปุ่มใหญ่ ใช้งานจอสัมผัสได้ดี เหมาะกับไลน์คัดแยกหรือโรงงาน

ความเป็นส่วนตัว

ทุกอย่างประมวลผลฝั่งผู้ใช้ในเดโมนี้ ไม่มีการอัปโหลดไฟล์ขึ้นเซิร์ฟเวอร์

พร้อมต่อยอด API

โครงหน้าบ้านแยกชัดเจน รองรับการเชื่อมต่อ API inferencing ได้ในอนาคต

เข้าถึงได้ (A11y)

ใช้ semantic HTML, ป้ายกำกับชัดเจน, รองรับคีย์บอร์ด และสกรีนรีดเดอร์

ทำงานอย่างไร

สำหรับโปรดักชันจริง มักใช้สายพาน + กล้อง + โมเดล CNN/Transformer ตรวจจับตำหนิ (เชื้อรา, แตกหัก, สีผิดปกติ) จากนั้นคัดแยกด้วยโซลินอยด์ลม/แขนกล

1) ถ่ายภาพ

แสงสม่ำเสมอ ฉากหลังตัดกัน (เช่น สีดำ/ขาว) ลดเงาและสะท้อนแสง

2) วิเคราะห์ด้วย AI

จำแนก ผ่าน/ไม่ผ่าน, โลเคชันตำหนิ และคะแนนความมั่นใจ

3) คัดแยกอัตโนมัติ

เชื่อม PLC/หุ่นยนต์ สั่งดีดเมล็ดที่ไม่ผ่านออกจากสายพาน

คำถามที่พบบ่อย

เดโมนี้ใช้ AI จริงหรือไม่?

เป็นการจำลองฝั่งหน้าเว็บเพื่อสาธิต UI/UX เท่านั้น สำหรับระบบจริงควรเชื่อมต่อ API โมเดล (เช่น ONNX/TensorRT/TF Serving) ผ่าน HTTPS

รองรับการใช้งานในโรงงานอย่างไร?

ปรับ UI ให้คอนทราสต์สูง ปุ่มใหญ่ และเพิ่มโหมดออฟไลน์ + PWA เพื่อลดการพึ่งพาเครือข่าย

ความเป็นส่วนตัวของรูปภาพ?

เดโมนี้ประมวลผลในเบราว์เซอร์ ไม่ส่งไฟล์ไปที่เซิร์ฟเวอร์ และสามารถกด “ล้างทั้งหมด” เพื่อลบข้อมูลจากหน้านี้ได้ทันที
